# Cognitive Accessibility Skill

## Overview
This skill provides guidance on designing and developing digital products that are accessible to people with a wide range of cognitive abilities.

## Purpose
The purpose of this skill is to create digital experiences that are easy to understand and use for everyone, including people with learning disabilities, attention deficit disorders, and other cognitive differences.

## When to Use
Use this skill when you want to:
- Design a new product or feature that is cognitively accessible.
- Audit an existing product for cognitive accessibility.
- Improve the user experience for people with cognitive disabilities.

## Key Features
- **Cognitive Accessibility Checklist**: A comprehensive checklist covering clear language, simple navigation, and other design considerations.
- **Best Practices**: Guidance on writing in plain language, using consistent layouts, and providing clear instructions.
- **Personas**: A set of personas representing users with different cognitive abilities.

## Usage Examples

### Example 1: Auditing a Website
```
Audit https://example.com for cognitive accessibility.
```
