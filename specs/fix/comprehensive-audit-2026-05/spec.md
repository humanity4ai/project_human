# Comprehensive Audit: All Prompt-Templates, Skills, MCPs

Status: Draft
Version: 1.0
Last updated: 2026-05-25

## Overview

Complete audit of every component generated by the `humanity4ai/project_human` repository:
10 SKILL.md files, 10 skill.yaml files, 10 MCP handlers, 20 JSON schemas, 10 sets of Python
scripts, 2 LLM discovery files (llms.txt, llms-full.txt), SYSTEM_PROMPT.md, knowledge-core,
eval system, and all documentation. 42 issues found across 6 domains.

### Findings

## Acceptance Criteria

### Epic 1: Fix Critical Handler Safety Gaps [MUST]

AC-1.1: grief_support_response detects crisis in message [MUST]
Given grief handler receives a message containing crisis signals (suicidal, self-harm, hopelessness)
When the handler processes the message
Then escalation_guidance contains at least 4 crisis resource items (including 988, 741741)
And care_notes includes a crisis-specific warning

AC-1.2: supportive_reply auto-assesses risk from message [MUST]
Given supportive_reply receives a message with crisis signals
When risk_level is not explicitly "high" by caller
Then the handler auto-escalates to high risk
And escalation_guidance includes full crisis resources

AC-1.3: supportive_reply locale has functional impact [SHOULD]
Given supportive_reply receives locale="en-GB"
When reply is generated
Then UK-specific crisis resources are used (Samaritans 116 123)
Not generic/international resources

### Epic 2: Unify Pattern Sets [MUST]

AC-2.1: Depression handler uses ALL Python + TS patterns [MUST]
Given 34+ patterns exist in Python scripts but not TS handler
When patterns.ts is expanded
Then all depression patterns from Python scripts are added to patterns.ts
And handler checks ALL categories (stigma, medical, minimizing, clinical, plus existing UX patterns)

AC-2.2: Grief handler analyzes message content [MUST]
Given 22+ patterns exist in Python scripts
When grief handler is updated
Then message is analyzed for cliches, minimizing, crisis, overwhelm patterns
And crisis detection is called on message content
And care_notes reflect detected patterns

AC-2.3: Supportive handler analyzes message content [MUST]
Given 85+ patterns exist in Python scripts
When supportive_reply is updated
Then message is analyzed for crisis signals, emotion cues, and boundary violations
And risk_level is auto-adjusted from "low" to "high" when ≥ 2 crisis patterns are detected
And reply text acknowledges the specific detected emotion when present (e.g., mentions "anxiety" if anxiety cues detected)
